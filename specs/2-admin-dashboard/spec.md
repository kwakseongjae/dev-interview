## 1. Executive Summary

비회원 및 회원의 면접 세션 사용 데이터를 추적하고 분석할 수 있는 관리자 전용 대시보드를 제공합니다. 비회원 사용 패턴 파악, 전환율 분석, 서비스 개선 인사이트 도출을 목적으로 합니다.

### Business Value

- **데이터 기반 의사결정**: 비회원 사용 패턴을 정량적으로 파악하여 서비스 개선 방향 도출
- **전환율 최적화**: 비회원 → 회원 전환 퍼널을 분석하여 가입 유도 전략 수립
- **서비스 건강성 모니터링**: 일별/주별 사용 현황을 한눈에 확인하여 이상 징후 조기 감지

## 2. User Stories & Acceptance Criteria

### US-1: 관리자 대시보드 접근 (P1)

**As a** 서비스 관리자
**I want to** 관리자 전용 대시보드에 접근하고 싶다
**So that** 서비스 사용 현황을 한눈에 파악할 수 있다

**Acceptance Criteria**:

- [ ] 관리자 이메일로 로그인한 사용자만 대시보드에 접근할 수 있다
- [ ] 비관리자가 대시보드 URL에 접근하면 홈페이지로 리다이렉트된다
- [ ] 미로그인 사용자가 접근하면 홈페이지로 리다이렉트된다
- [ ] 관리자 목록은 서버 환경변수로 관리된다

### US-2: 핵심 지표 확인 (P1)

**As a** 서비스 관리자
**I want to** 서비스의 핵심 사용 지표를 한눈에 확인하고 싶다
**So that** 서비스 현황을 빠르게 파악할 수 있다

**Acceptance Criteria**:

- [ ] 총 세션 수가 숫자 카드로 표시된다
- [ ] 오늘의 세션 수가 표시된다
- [ ] 총 사용자 수가 표시된다
- [ ] 비회원 세션 비율이 백분율로 표시된다
- [ ] 각 지표는 선택한 기간에 맞게 업데이트된다

### US-3: 일별 세션 추이 확인 (P1)

**As a** 서비스 관리자
**I want to** 일별 세션 추이를 차트로 확인하고 싶다
**So that** 사용량 증감 트렌드를 파악할 수 있다

**Acceptance Criteria**:

- [ ] 일별 세션 수가 라인 차트로 표시된다
- [ ] 비회원과 회원 세션이 구분되어 표시된다 (색상 구분)
- [ ] 차트에 마우스를 올리면 해당 날짜의 상세 수치가 툴팁으로 표시된다
- [ ] 기간 필터에 따라 차트 범위가 변경된다

### US-4: 인기 쿼리 확인 (P1)

**As a** 서비스 관리자
**I want to** 비회원이 가장 많이 검색한 쿼리를 확인하고 싶다
**So that** 비회원들의 관심사와 수요를 파악할 수 있다

**Acceptance Criteria**:

- [ ] 비회원 기준 인기 쿼리 Top 10이 순위와 함께 표시된다
- [ ] 각 쿼리의 사용 횟수가 함께 표시된다
- [ ] 선택한 기간 내의 데이터만 반영된다

### US-5: 면접 유형별 분포 확인 (P1)

**As a** 서비스 관리자
**I want to** 면접 유형별 세션 분포를 확인하고 싶다
**So that** 인기 있는 면접 유형을 파악하고 콘텐츠 전략을 수립할 수 있다

**Acceptance Criteria**:

- [ ] 면접 유형별 세션 수가 차트(바 차트)로 표시된다
- [ ] 각 유형의 이름과 색상이 구분되어 표시된다
- [ ] 기간 필터에 따라 데이터가 변경된다

### US-6: 기간 필터 (P1)

**As a** 서비스 관리자
**I want to** 데이터 조회 기간을 선택하고 싶다
**So that** 원하는 기간의 데이터만 집중적으로 분석할 수 있다

**Acceptance Criteria**:

- [ ] 최근 7일, 30일, 전체 중 선택할 수 있다
- [ ] 기간 선택 시 모든 지표와 차트가 해당 기간으로 업데이트된다
- [ ] 기본값은 최근 30일이다
- [ ] 기간 변경 시 로딩 상태가 표시된다

### US-7: 비회원 → 회원 전환율 확인 (P2)

**As a** 서비스 관리자
**I want to** 비회원이 회원으로 전환하는 비율을 확인하고 싶다
**So that** 가입 유도 전략의 효과를 측정할 수 있다

**Acceptance Criteria**:

- [ ] 비회원 세션 중 회원으로 전환(claim)된 세션의 비율이 표시된다
- [ ] 전환된 세션 수와 전체 비회원 세션 수가 함께 표시된다

### US-8: 세션 완료율 비교 (P2)

**As a** 서비스 관리자
**I want to** 비회원과 회원의 세션 완료율을 비교하고 싶다
**So that** 가입이 학습 지속성에 미치는 영향을 파악할 수 있다

**Acceptance Criteria**:

- [ ] 비회원 세션 완료율과 회원 세션 완료율이 나란히 표시된다
- [ ] 백분율과 함께 시각적 표시(프로그레스 바 등)가 제공된다

### US-9: 최근 비회원 세션 목록 (P2)

**As a** 서비스 관리자
**I want to** 최근 비회원 세션 목록을 확인하고 싶다
**So that** 비회원들의 구체적인 사용 패턴을 파악할 수 있다

**Acceptance Criteria**:

- [ ] 최근 비회원 세션이 테이블 형태로 표시된다
- [ ] 각 세션의 쿼리, 생성 시간, 질문 수, 완료 여부가 표시된다
- [ ] 최대 20개 세션이 표시된다

## 3. Functional Requirements

### FR-1: 관리자 인증 (P1)

관리자 식별은 서버 환경변수에 등록된 이메일 허용 목록으로 수행한다.

- 관리자 페이지 접근 시 서버 사이드에서 인증 검증
- 관리자 API 호출 시에도 동일한 인증 검증 수행 (방어적 이중 체크)
- 인증 실패 시 홈페이지로 리다이렉트 (에러 메시지 없음, 조용한 리다이렉트)

### FR-2: 통계 데이터 조회 (P1)

단일 API 엔드포인트에서 모든 대시보드 데이터를 반환한다.

- 기간 파라미터: 7일(`7d`), 30일(`30d`), 전체(`all`)
- 반환 데이터: 개요 지표, 일별 추이, 인기 쿼리, 유형 분포, 전환율, 완료율, 최근 세션
- 데이터 집계는 서버 사이드에서 수행 (단일 DB 라운드트립)

### FR-3: 대시보드 시각화 (P1)

- KPI 카드: 총 세션, 오늘 세션, 총 사용자, 비회원 비율 (4개)
- 라인 차트: 일별 세션 추이 (비회원/회원 구분)
- 바 차트: 면접 유형별 분포
- 리스트: 인기 쿼리 Top 10
- 테이블: 최근 비회원 세션 (최대 20개)

### FR-4: 기간 필터 (P1)

- 3개 프리셋: 최근 7일, 최근 30일, 전체
- 기본값: 최근 30일
- 필터 변경 시 모든 위젯 동시 업데이트

### FR-5: 전환율 및 완료율 (P2)

- 전환율: (claim된 비회원 세션 수 / 전체 비회원 세션 수) × 100
- 완료율: 비회원과 회원 각각의 (완료 세션 수 / 전체 세션 수) × 100

## 4. Non-Functional Requirements

### NFR-1: 성능

- 대시보드 초기 로딩 3초 이내
- 기간 필터 변경 시 2초 이내 업데이트

### NFR-2: 보안

- 관리자 인증은 서버 사이드에서만 수행 (클라이언트 우회 불가)
- 통계 API는 관리자만 호출 가능
- 개인 식별 정보(이메일 등)는 대시보드에 노출하지 않음

### NFR-3: 접근성

- 대시보드는 반응형 레이아웃으로 모바일에서도 사용 가능
- 로딩 중 스켈레톤 UI 표시

## 6. Key Entities

### 관리자 (Admin)

- 환경변수에 등록된 이메일을 가진 인증된 사용자
- 대시보드 및 통계 API 접근 권한 보유

### 면접 세션 (Interview Session)

- 비회원 세션: `user_id`가 NULL인 세션
- 회원 세션: `user_id`가 존재하는 세션
- Claim된 세션: 비회원으로 생성 후 회원으로 전환(claim)된 세션

### 통계 데이터 (Stats)

- 개요 지표: 총 세션, 오늘 세션, 총 사용자, 비회원 비율
- 시계열 데이터: 일별 세션 수 (비회원/회원 구분)
- 분포 데이터: 면접 유형별 세션 수
- 전환 데이터: 비회원 → 회원 전환율, 세션 완료율

## 7. Assumptions

- 관리자는 1~2명으로 소규모이며, 환경변수 기반 관리가 적합하다
- 기존 `interview_sessions` 테이블의 `user_id = NULL` 조건으로 비회원 세션을 식별할 수 있다
- Claim된 세션은 원래 `user_id = NULL`이었다가 이후 `user_id`가 설정된 세션으로 판단한다 (별도 `claimed_at` 컬럼이 있다고 가정, 없으면 추가 필요)
- 대시보드는 실시간 업데이트가 필요하지 않으며, 페이지 로딩/필터 변경 시 데이터를 조회한다
- 비회원 세션의 개인 식별 정보(IP, 브라우저 등)는 추적하지 않는다

## 8. Scope Boundaries

### 포함 (In Scope)

- 관리자 대시보드 페이지 및 인증
- 비회원/회원 세션 통계 (개요, 추이, 분포)
- 인기 쿼리 Top 10
- 기간 필터 (7일, 30일, 전체)
- 전환율 및 완료율
- 최근 비회원 세션 목록

### 제외 (Out of Scope)

- 실시간 알림/모니터링
- 관리자 역할 관리 UI (환경변수로 관리)
- 데이터 내보내기 (CSV/Excel)
- 개별 사용자 분석 (코호트 분석)
- A/B 테스트 연동
- 관리자 행위 감사 로그
